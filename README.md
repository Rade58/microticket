**DAKLE IMAJ NA UMU DA TI JE DATBASE GDE SU TI USERI SAMO VEZAN ZA `auth` MICROSERVICE**

*TAJ MALICIOUS USER, MOZE NA PRIMER DA NAPRAVI REQUEST DO NEKOG TVOG `orders` ILI `purchase` MICROSERVICE-A*

**S OBZIROM DA IMA TOKEN ILI COOKIE, TAJ SERVICE CE KORISTITI TU JWT ILI SESSION COOKIE AUTHENTICATION LOGIKU, KAKO BI PROVERIO DA LI JE TAJ KORISNIK AUTHETICATED**

# REMINDER OF COOKIES MECHANISM AND JWT MECHANISM

DAKLE ZELI MDA SE PODSETIM NA OVA DVE STVARI, KOJE IMAJU NEKIH SLICNOSTI, ALI SE IPAK FUNDAMANTALNO RAZLIKUJU

## COOKIES

1. DAKLE IZ BROWSER-A ODNOSNO CLIENT-A SE SALJE REQUEST PREM NEKOM SERVER-U

2. KADA SERVER SALJE RESPONSE NAZAD DO CLIENT-A, MOZE OPCIONO INCLUDE-OVATI HEADER, KOJI SE ZOVE `Set-Cookie` I KOJI IMA NEKU VREDNOST `'ssdfdgdfhfnv1'` (VALUE MOZE BITI STRING KOJI CONTAIN-UJE ANY INFORMATION THAT YOU WANT, NEKI ARBITRARY INFO KOJI MOZES SMISLITI, **BILO STA** (I TO JE KLJUCNA STAV DA SE REMEMBER-UJE ABOUT COOKIES IN GENERAL))

**I TAJ INFORMATION CE AUTOMATSKI BITI STORED IN BROWSER**

3. KADA BROWSER OPET NAPRAVI FOLLOW UP REQUEST, DO ISTOG DOMAINE, ODNOSNO ISTOG SERVER-A; BROWSER CE SE POSTARATI DA UZEM TAJ LITTLE PIECE OF INFORMATION, I APPEND-UJE GA NA REQUEST KAO HEADER `Cookie`

## JWT

UZIMAM NEKI ARBITRARY PIECE OF INFORMATION, KAO STO JE OVAJ OBJEKAT NAPRAVLJEN OD NEKIH INFORMACIJA USER-A: `{userId:'fghgf4', favoriteCar: 'BMW'}`

TAJ INFORMATION OBICNO NAZIVAMO PAYLOAD-OM

ONDA TIM PAYLOAD-OM NAHRANIMO `JWT CREATION ALGORITHM`

A TAJ ALGORITAM CE DA ISPLJUNE NAS JASON WEB TOKEN, KOJI PREDSTAVLJA NEKI SET KARAKTERA; NEKAKV ENCODED STRING

**ONO STO JE BITNO JESTE DA JE U TOM TOKENU STORED ONAJ ORIGINAL PAYLOAD**

TAJ STRING SE MOZE LAKO UZETI I THROW-OVATI U NEKI `JWT DECODING ALGORITHM`, I NA TAJ NACIN BI EXTRACT-OVAO ONAJ ORIGINAL PAYLOAD

POSTOJI NEKOLIKO NACINA DA SE POSALJE TOKEN OD CLIENT-A:

1. CLIENT BI POSLAO TOKEN KAO VREDNOST `Authorization` HEADERA, ALI BI BREDNOST BILA U FORMATU: `Bearer ${Your Token}`, STO SAM JA RADIO U NEKIM RANIJIM PROJEKTIMA

2. MOZEMO SLATI TOKEN U BODY-JU REQUESTA; TO NARVNO MORA BITI POST, PUT AND SO ON

3. I MOZEMO SLATI TOKEN KAO VREDNOST `Cookie` HEADER-A, CIME BI COOKIE BIO MANAGED BU THE BROWSER I BIO BI SENT BY ANY FOLLOW UP REQUEST

# COOKIE JESTE `TRANSPORT MECHANISM` DOK JE JWT, USTVARI `AUTHENTICATION/AUTHORIZATION MECHANISM`

**COOKIE MOVE-UJE ANY KIND OF DATA BETWEEN BROWSER AND SERVER**

**COOKIE JE AUTOMATICALLY MANGED BY THE BROWSER, STO ZNACI DA CE BITI INCLUDED ON ANY FOLLOWUP REQUEST**

**JWT STORE-UJE ANY DATA THAT YOU WANT**

**ALI MORAS DA GA MANGE-UJES MANUALLY**
